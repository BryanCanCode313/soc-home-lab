# Incident Reports

This folder contains incident reports created while investigating **simulated security events** in my Splunk SOC home lab.

Each investigation follows a structured incident response workflow designed to build the skills expected of a Tier 1 SOC Analyst. Rather than simply configuring security tools, these reports demonstrate how security events are analyzed, documented, and assessed using evidence collected from Sysmon and Splunk.

## Each report includes

* Scenario or alert summary
* Investigation objective
* Evidence collected from Splunk and Sysmon
* Analysis of the observed activity
* Assessment (Benign or Suspicious)
* Supporting screenshots
* Lessons learned (when applicable)

## Purpose

The purpose of these reports is to demonstrate practical SOC analyst skills, including:

* Investigating Windows process creation events
* Searching and filtering logs using Splunk SPL
* Interpreting Sysmon event data
* Distinguishing legitimate activity from suspicious behavior
* Writing clear, evidence-based incident documentation

> **Note:** All investigations were performed in a controlled home lab environment using simulated scenarios for educational and portfolio purposes.
